# 전문적이고 자연스러운 기술 블로그 작성 원칙 (Human-Centric Tech Writing)

## 🎯 핵심 철학: "Professional Insight, Human Tone"
AI처럼 보이지 않는 글은 **'구체성'**과 **'리듬감'**에서 옵니다. 독자가 마치 사수(Senior Developer)에게 직접 설명을 듣는 듯한 느낌을 주어야 합니다.

---

## 🛠️ 구체적 실천 지침

### 1. 문장 리듬감 (Sentence Pacing)
- **단장문 혼합**: 모든 문장을 길게 쓰지 마세요. 핵심은 짧게, 설명은 길게 섞습니다.
- **예시**: "Framer Motion은 강력합니다. 하지만 무턱대고 모든 요소에 적용하면 메인 스레드에 과부하를 줄 수 있습니다. 특히 복잡한 SVG 경로 애니메이션을 다룰 때는 더욱 주의가 필요합니다."

### 2. 기계적인 연결어 제거 (Natural Transitions)
- **사용 금지**: "첫째", "둘째", "또한", "게다가", "결론적으로"
- **권장 방식**: 문맥을 이어가는 질문이나 상황 제시를 사용하세요.
  - "그렇다면 실무에서는 이 값을 어떻게 설정해야 할까요?"
  - "이 문제를 해결하기 위해 우리가 주목해야 할 지점은 바로..."
  - "재미있는 점은, 대부분의 개발자가 이 부분에서 실수를 한다는 것입니다."

### 3. '진짜' 개발자의 현장 언어 사용 (Developer Nuance)
- **추상적 표현 지양**: "매우 유용합니다", "성능이 좋아집니다" (X)
- **구체적 현장 묘사**: "z-index가 꼬여서 모달이 뒤로 숨는 현상", "Lighthouse 점수가 빨간색으로 변하는 고통", "배포 직전에 발견되는 레이아웃 깨짐" (O)
- **전문 용어의 자연스러운 노출**: 리플로우(Reflow), 트리 쉐이킹(Tree Shaking), 관성 애니메이션 등 기술 용어를 설명 없이 적절히 섞어 전문성을 높입니다.

### 4. 솔직한 한계 인정 (Acknowledging Limits)
- 완벽함을 주장하는 AI와 달리, 사람은 도구의 한계를 압니다.
- **예시**: "이 도구가 모든 문제를 해결해주지는 않습니다. 하지만 반복적인 CSS 타이핑 시간을 80% 이상 줄여주는 것만큼은 확실합니다."

### 5. 능동적이고 확신 있는 어조 (Active Voice)
- "~인 것으로 생각됩니다", "~해볼 수 있습니다" 같은 추측성 표현을 줄이세요.
- **예시**: "경험상 이 수치가 가장 안정적입니다.", "이 방식이 Next.js 환경에서 권장되는 표준입니다."

## 🎨 시각적 가독성 원칙 (Visual Readability)
- **배경-글자 대비**: 텍스트가 배경에 묻히지 않도록 항상 충분한 명도 대비를 확보합니다. (검은 배경엔 밝은 회색/흰색, 흰 배경엔 짙은 회색/검정)
- **정보의 위계**: 중요한 정보는 굵게(bold) 하거나 색상을 주되, 한 페이지에 너무 많은 강조색을 쓰지 않습니다.
- **행간과 자간**: 글이 빽빽해 보이지 않도록 `leading-relaxed`와 같은 여유로운 행간을 Tailwind로 적용합니다.

## 🚫 AI 티를 내는 '결정적 증거' 제거 (Strict)

| AI의 특징 | 사람의 특징 (수정 방향) |
|:---|:---|
| "여러분 안녕하세요" 등 불필요한 인사 | 바로 본론으로 들어가며 상황 제시 |
| "요약하자면", "결론적으로" | "정리하자면 이 도구의 핵심은..." 또는 자연스러운 마무리 |
| "~해보세요", "~해볼까요?" (선생님 톤) | "~하는 방법이 효율적입니다", "~하면 해결됩니다" (사수 톤) |
| "놀라운", "혁신적인", "완벽한" 등 수식어 과다 | 수식어 대신 "빌드 속도를 2초 줄여주는" 등 결과 수치 제시 |

## ✅ 자가 점검 (Writing Check)
- [ ] 문장이 '첫째, 둘째'로 시작하지 않는가?
- [ ] '여러분'이나 '해보세요'라는 표현이 들어있지 않은가?
- [ ] 개발자가 공감할 만한 구체적인 불편함(Pain Point)이 묘사되었는가?
- [ ] 사수가 옆에서 코드 리뷰를 해주는 듯한 톤인가?
- [ ] '혁신적인' 같은 추상적인 수식어가 제거되었는가?
