# Step 8: 자동 단위 테스트 생성 (Vitest)

## 🎯 이 단계의 목표
`lib/` 폴더에 분리된 비즈니스 로직에 대한 **유닛 테스트**를 작성하여 도구의 신뢰성을 확보합니다.

---

## 📋 수행 작업

### 1. 로직 분석
- `lib/use-logic.ts` 또는 관련 유틸리티 파일의 함수들을 분석합니다.
- 입력값에 따른 출력값, 에러 케이스, 경계값(Edge Cases)을 도출합니다.

### 2. 테스트 코드 작성 (`__tests__` 폴더)
- Vitest 프레임워크를 사용합니다.
- `describe`, `it`, `expect`를 사용하여 가독성 있게 구조화합니다.

---

## 🛠️ 작성 규칙
- **순수 함수 테스트**: UI 렌더링 없이 로직 자체의 정확성만 검증합니다.
- **Mocking**: 외부 API나 복잡한 의존성이 있다면 `vi.mock()`을 활용합니다.
- **파일명**: `logic.test.ts` 또는 `[도구명].test.ts`

---

## ✅ 완료 기준
- `src/features/tools/tools/[slug]/__tests__/` 폴더에 테스트 파일이 생성됨.
- `npm test [slug]` 명령 실행 시 모든 테스트가 통과함.
