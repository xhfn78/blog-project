# Step 6-3: 하위 컴포넌트 분리 구현

## 🎯 이 단계의 목표
거대해질 수 있는 메인 페이지를 작은 기능 단위의 컴포넌트로 쪼개어 `ui/` 폴더에 구현합니다.

---

## 📋 수행 작업

### 1. 하위 컴포넌트 도출
메인 UI를 최소 2개 이상의 의미 있는 단위로 나눕니다.
- 예: `ControlPanel.tsx` (사용자 입력부)
- 예: `PreviewDisplay.tsx` (결과 시각화부)
- 예: `HistoryList.tsx` (이전 기록)

### 2. 개별 컴포넌트 구현
- `src/shared/ui`의 컴포넌트들을 적극 활용하여 구현합니다.
- 각 하위 컴포넌트는 `model/types.ts`에서 정의한 타입을 공유합니다.

---

## 🛠️ 구현 가이드
- **컴포넌트화 원칙**: 한 파일이 150줄을 넘지 않도록 노력하세요. 넘을 것 같으면 더 작은 단위로 쪼개세요.
- **Props 전달**: 상위에서 로직(State, Handler)을 Props로 받아 단순히 렌더링하는 'Presentational Component' 형태로 작성합니다.

---

## ✅ 완료 기준
- `src/features/tools/tools/[slug]/ui/` 폴더에 최소 2개 이상의 독립적인 컴포넌트 파일이 생성됨.
