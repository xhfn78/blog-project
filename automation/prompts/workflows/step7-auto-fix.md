# Step 7: 품질 검증 및 자동 수정

## 🎯 이 단계의 목표
`validate-tool.js`의 검증 결과를 바탕으로 생성된 도구의 오류와 경고를 해결하여 **최종 배포 가능 수준**으로 품질을 높입니다.

---

## 📋 수행 작업

### 1. 검증 리포트 분석
`node automation/scripts/validate-tool.js <slug> --verbose` 명령의 결과를 읽고 다음을 분류합니다.
- **Errors (❌)**: 반드시 수정해야 하는 치명적 결함 (파일 누락, 타입 오류 등)
- **Warnings (⚠️)**: SEO나 UX 향상을 위해 권장되는 수정 사항 (글자 수 부족, 태그 개수 등)

### 2. 코드 수정 전략

#### **A. SEO 콘텐츠 부족 시**
- `index.tsx`의 설명 섹션을 보강합니다.
- 기술적 배경, 실무 시나리오, FAQ 섹션을 더 구체적인 예시와 함께 확장합니다.
- 단순히 양을 늘리는 것이 아니라, 사용자에게 실질적인 가치를 주는 정보를 추가합니다.

#### **B. AI 티 나는 표현 발견 시**
- `automation/prompts/rules/rule-natural-writing.md` 지침을 다시 복습합니다.
- "해보세요", "놀라운", "여러분" 등의 표현을 객관적이고 기술적인 문장으로 교체합니다.

#### **C. 설정 오류 (tool.config.ts) 시**
- description을 250자 이상의 풍부한 문장으로 재작성합니다.
- 관련도가 높은 태그를 6-8개로 조정합니다.

---

## 🔄 반복 루프
수정이 완료되면 다시 검증을 실행합니다.
```bash
node automation/scripts/validate-tool.js <slug>
```
**모든 에러가 사라지고 경고가 최소화될 때까지 이 과정을 반복합니다.**
