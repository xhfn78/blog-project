# Step 6-2: 도메인 로직 및 타입 정의

## 🎯 이 단계의 목표
도구의 핵심 비즈니스 로직과 TypeScript 타입을 독립적으로 구현하여 코드의 견고함을 확보합니다.

---

## 📋 수행 작업

### 1. 타입 정의 (`model/types.ts`)
- 도구에서 사용하는 모든 상태(State)와 Props의 인터페이스를 정의합니다.
- `framer-motion` 등의 외부 라이브러리 타입을 정확히 확장합니다.

### 2. 비즈니스 로직 분리 (`lib/` 폴더)
- 메인 컴포넌트에서 로직을 분리하여 Custom Hooks 또는 순수 함수(Pure Function)로 작성합니다.
- **예시**: 데이터 변환 공식, 복잡한 상태 업데이트 로직, 유효성 검사기 등

---

## 🛠️ 구현 가이드
- **관심사 분리**: 이 단계에서는 UI 디자인(CSS, JSX)을 절대 고민하지 마세요. 오직 **'데이터가 어떻게 변하는지'**에만 집중합니다.
- **테스트 용이성**: 순수 함수 위주로 작성하여 나중에 `__tests__`를 만들기 쉽게 합니다.

---

## ✅ 완료 기준
- `src/features/tools/tools/[slug]/model/` 폴더에 타입 파일이 생성됨.
- `src/features/tools/tools/[slug]/lib/` 폴더에 핵심 로직 파일이 생성됨.
