## 🎨 VibeVisual Standard 디자인 원칙 (Strict Readability)

### 1. 배경 및 텍스트 대비 (WCAG AA 준수)

> [!CAUTION]
> **가독성은 최우선 순위입니다.** 아래 색상 팔레트를 반드시 준수하세요.

#### 필수 색상 규격 (다크 모드 기준)

| 용도 | 권장 클래스 | 금지 클래스 | 대비율 |
|:---|:---|:---|:---:|
| **배경** | `bg-slate-900`, `bg-slate-950`, `bg-black` | `bg-slate-700` 이상 밝은 배경 | - |
| **제목** | `text-white`, `text-slate-100` | `text-gray-400` 이하 어두운 색 | 15:1+ |
| **본문** | `text-slate-200`, `text-slate-300` | `text-muted-foreground` 단독 사용 | 8:1+ |
| **강조** | `text-indigo-300`, `text-emerald-300`, `text-amber-300` | `text-indigo-500` 이하 어두운 강조색 | 7:1+ |
| **보조** | `text-slate-400` (최소 대비) | `text-slate-500` 이하 | 4.5:1+ |

#### 절대 금지 조합
- ❌ `text-muted-foreground` + `bg-slate-900` (대비 부족)
- ❌ `text-indigo-500` + `bg-slate-900` (대비 부족)
- ❌ `text-purple-500` + `bg-slate-900` (대비 부족)
- ❌ 밝은 배경 + 밝은 텍스트
- ❌ 어두운 배경 + 어두운 텍스트

#### 필수 규칙
1. **어두운 배경 + 밝은 텍스트** 조합만 허용
2. 모든 텍스트 대비율 **4.5:1 이상** 확보
3. 그라데이션 배경 위 텍스트: 반드시 `bg-black/40` 이상 오버레이 적용
4. 테이블 셀: `text-slate-200` 이상 사용

- **어두운 배경 (Dark)**: `bg-slate-900` 또는 `bg-slate-950` 사용
  - 제목: `text-white` (대비 15:1)
  - 본문: `text-slate-200` (대비 12:1)
  - 보조설명: `text-slate-400` (최소 대비 4.5:1)
- **강조 섹션 (Gradients)**: 텍스트 뒤에 어두운 오버레이(`bg-black/40`)를 필수 적용하여 배경색에 상관없이 가독성 확보.

### 2. 다이어그램 및 코드 블록
- **Mermaid/Code**: 반드시 `bg-black` 또는 `bg-slate-950` 배경을 가진 카드 내부에 배치.
- **텍스트 색상**: `text-slate-100` (화이트 계열) 또는 고대비 색상(`text-emerald-400`) 사용.

## 📐 표준 섹션 구조 (Fixed Template)

### 1. Hero 도입부
- 큰 타이포그래피 + 짙은 배경.
- `text-slate-100` 본문을 사용하여 한 눈에 들어오게 구성.

### 2. 작동 원리 (Mermaid)
- **반드시 어두운 배경 카드** 위에 배치.
- 흐릿한 색상 금지.

### 3. 기술 상세 비교 (Table)
- 표 헤더는 `bg-slate-800`, 셀 텍스트는 `text-slate-200` 고정.

## 📐 전체 구조 및 구성 요소

### 1. 도입부 (400자+)
- 도구의 탄생 배경과 실무에서의 고통(Pain Points) 언급.
- 이 도구가 왜 필요한지 당위성 부여.

### 2. 주요 기능 (500자+)
- 단순 나열이 아닌, 각 기능이 개발자의 생산성에 어떻게 기여하는지 기술.

### 3. 작동 원리 도식화 (Mermaid/SVG) - [필수]
- **Mermaid 코드 블록**을 사용하여 데이터의 흐름이나 로직의 순서를 시각화합니다.
- 예시:
  ```mermaid
  graph LR
    A[사용자 입력] --> B{파싱 엔진}
    B --> C[규격 변환]
    C --> D[최종 코드 출력]
  ```

### 4. 실무 활용 시나리오 (600자+)
- 3가지 이상의 구체적인 Before/After 사례 제시.

### 5. 기술적 배경 및 속성 비교 표 (700자+)
- 기술 표준(W3C, MDN 등) 인용.
- **HTML Table**을 사용하여 상세 속성값 비교 분석.

### 6. 자주 묻는 질문 (FAQ) - [필수 5개 이상]
- `<h3>Q1: 질문 내용</h3>` 형식을 엄격히 준수하여 검증기 통과.
