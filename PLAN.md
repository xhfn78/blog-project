## 1단계: 블로그 기능 제거 및 홈페이지 변환 계획 (남은 부분)

다음은 블로그 기능 제거 및 홈페이지 변환 1단계의 남은 작업에 대한 순차적인 계획입니다. 모든 지침은 순수 자연어로 작성되었으며, 코드 스니펫은 포함되지 않습니다.

### 1. `src/app/api/content/route.ts` 검토 및 블로그 관련 API 엔드포인트 제거
*   **목표:** `src/app/api/content/route.ts` 파일을 열어 해당 파일에 블로그 콘텐츠(예: 게시물 생성, 조회, 삭제)와 관련된 API 엔드포인트 로직이 포함되어 있는지 확인합니다.
*   **작업 내용:** 만약 블로그 관련 로직이 발견되면, 해당 로직을 파일에서 제거합니다. 이는 블로그 기능이 더 이상 필요 없으므로, 관련 API도 함께 정리하는 과정입니다.

### 2. `src/app/page.tsx` 리팩토링 (홈페이지 변환)
*   **목표:** 현재 메인 페이지(`src/app/page.tsx`)의 내용을 제거하고, 대신 모든 도구 목록을 직접 표시하도록 페이지를 완전히 재구성합니다. 이 과정에서 "블로그 보러가기" 또는 "도구 사용하러 가기"와 같은 기존의 어떤 버튼도 페이지에 남아있지 않도록 합니다.
*   **작업 내용:**
    *   `src/app/page.tsx` 파일의 모든 내용을 비웁니다.
    *   도구 목록을 가져와서 표시하는 새로운 로직으로 대체합니다. 이 로직은 기존 `src/app/tools/page.tsx`에서 도구 목록을 가져오는 방식을 참고할 것입니다.
    *   도구 목록을 표시하기 위해 `src/features/tools/ui/tool-grid.tsx` 컴포넌트를 재활용합니다. 이 컴포넌트는 모든 사용 가능한 도구를 깔끔하게 그리드 형태로 보여줄 것입니다.

### 3. `src/app/layout.tsx` 메타데이터 업데이트 (블로그 관련 항목 제거)
*   **목표:** `src/app/layout.tsx` 파일 내의 전역 메타데이터(`metadata` 객체)에서 블로그와 관련된 모든 설명, 키워드, Open Graph, Twitter Card 항목들을 찾아 제거하거나 수정합니다.
*   **작업 내용:**
    *   `metadata` 객체 안에서 "블로그"라는 단어가 포함된 `description`, `keywords`, `openGraph.description`, `twitter.description` 등의 속성을 검토합니다.
    *   블로그 관련 문구나 키워드를 제거하거나, 사이트의 새로운 정체성인 "도구 모음"에 맞춰 수정합니다.

이러한 작업을 통해 블로그 기능은 완전히 제거되고, 홈페이지는 순수하게 도구 모음의 허브 역할을 하게 됩니다.
